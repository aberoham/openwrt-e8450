OpenWrt E8450 Fleet Management - Changelog
==========================================

2026-04-03: IPv6 Prefix Delegation Restored (DUID Fix) + DNS Cleanup
----------------------------------------------------------------------
- Fixed DHCPv6 prefix delegation by setting DUID to DUID-LL (type 3)
  matching the original ISP-supplied router
- Root cause: OpenWrt 25.12 changed DUID generation from per-interface
  DUID-LL (type 3, MAC-derived) to a global DUID-UUID (type 4). The
  BNG only serves PD to the DUID in its lease database, which had the
  old DUID-LL format. This was introduced by an Oct 2025 commit and
  affects anyone upgrading from 24.10 to 25.12.
- Enabled masq6 on the tailscale firewall zone for IPv6 exit node
- Disabled Tailscale MagicDNS on router (was hijacking resolv.conf
  with SERVFAIL responses); router now uses its own dnsmasq
- Set peerdns=0 on IPv4 WAN to ignore ISP DNS consistently
- Upstream DNS: Google Public DNS (8.8.8.8/8.8.4.4 + IPv6 equivalents)
- See notes/ipv6-duid-fix.md for full technical details

2026-04-02: Major Upgrade to OpenWrt 25.12.2
---------------------------------------------
- Upgraded both APs from OpenWrt 24.10.6 to 25.12.2
- Kernel 6.6.127 → 6.12.74
- Tailscale 1.80.3 → 1.94.1 (Go 1.23 → Go 1.26)
- Package manager changed from opkg to apk
- Motivation: unblock Tailscale past 1.80.3 (24.10 Go version wall)
  and address likely cause of March 20 kernel crash
- See notes/tailscale-custom-build.md for research on the Go/Tailscale
  version constraint and custom build plan

2026-03-31: Downstairs AP Recovery from pstore Recovery Boot
------------------------------------------------------------
- Downstairs router kernel-crashed on 2026-03-20 ~17:09
- U-Boot pstore check detected crash data → auto-booted recovery initramfs
- Recovery mode runs from RAM with no overlay (default config, 192.168.1.1)
- All config/overlay data was intact on rootfs_data (UBIFS volume 7)
- Fix: cleared pstore entries, rebooted → normal production boot restored
- Crash logs saved to private/logs/crash-2026-03-20/
